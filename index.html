This specification document refers to the standards, patterns and examples used for the Verifiable Credential issuance and verification of the educational records requested by a post-secondary institution in Canada. These credentials are issued based on the JSON-LD format as defined by W3C which are cryptographically signed and provide a proof of authenticity. Credentials are issued through a platform (i.e. MyCredsTM) and verified by various industry partners to enable different services for the students or learners in Canada. This specification provides some reference material, the vocabulary used and the code samples to help the developers community in utilizing the academic credentials.

Comments regarding all aspects of this document are welcome. Please file issues directly on GitHub

Introduction

This specification describes a Linked Data vocabulary for asserting Verifiable Credentials related to an academic Micro-credential. This vocabulary is intended to be used in conjunction with the Verifiable Credentials Data Model 1.0 [[?VC-DATA-MODEL]] specification.

Use Cases and Requirements

There are various use cases that might be helpful for the readers in understating the MyCredsTM credential ecosystem. Existing use case covers the JSON-LD format and a basic pattern in issuing and verifying the micro-credentials of an academic record. The university or college accesses the MyCredsTM platform and request to issue a Verifiable Credential to the student or learner. The claim of this credential holds the information related to academic certificate, credential owner details and the institution issuing these Credentials. The use case requirements should be able to cover the the following.

The Academic Micro-credential Award Vocabulary

This vocabulary assumes all terms specified in the base Verifiable Credentials [[VC-DATA-MODEL]] context. In addition, the following classes are available for specifying information related to a micro-credential award.

Class Description
AcademicAward Specifies that the credential is a micro-credential award.

This document also specifies the following terms for expressing a micro-credential award:

Term Description
givenName The given name of the person who was awarded the micro-credential.
familyName The family name of the person who was awarded the micro-credential.
birthDate The date on which a individual was born.
email The email address of the recipient of the micro-credential.

The credential contains the following terms to express the award Details about the issued micro-credential:

Term Description
awardId The unique identifier of the micro-credential award.
awardTitle The name of the micro-credential award.
awardType The type of micro-credential award.
awardDescription A description of the micro-credential award.
honours The honours associated with the micro-credential award.
studyArea The study area associated with the micro-credential award.
dateAwarded The date on which the micro-credential was awarded.
dateIssued The date on which the micro-credential was issued.

The credential also contains the following terms to express the educational organization that issued the micro-credential:

Term Description
name The name of the educational organization.
url The URL of the educational organization.
identifier The identifier of the educational organization.
countryCode The country code of the educational organization.
